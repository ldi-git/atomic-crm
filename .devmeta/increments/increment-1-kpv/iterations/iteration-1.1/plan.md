# Iteration 1.1 ‚Äî Seeder scaffold + Makefile

**Goal:** Get the seeder plumbing in place. A `make seed` target that resets the local DB and runs a stub seeder which (a) verifies the connection is local, (b) verifies the DB is clean, (c) writes one smoke-test row to prove the round-trip works. No CRM data yet ‚Äî that's iteration 1.3. No auth users yet ‚Äî that's iteration 1.2.

## Deliverables

1. `tsx` and `@supabase/supabase-js` added as devDependencies (supabase-js is currently only a transitive dep via `ra-supabase-core` ‚Äî we want a stable direct dep for the script).
2. `scripts/seed-supabase.ts` with:
   - Hard guard: exit non-zero unless `SUPABASE_URL` is `127.0.0.1` / `localhost` (read from `.env*` files the same way Vite does, or via a config helper ‚Äî see Implementation below).
   - Service-role key resolved by `execa('npx', ['supabase', 'status', '-o', 'json'])` and parsing the response. No env-var setup required from the developer.
   - Build a `supabase-js` client with the service-role key.
   - Clean-DB precondition: query `companies`; if rows > 0, exit non-zero with a clear "DB not clean ‚Äî run `make seed` (which resets first) instead" message.
   - Smoke-test insert: write one row into `tags` (`name='__seed_smoke_test__', color='#000000'`). Easy to verify; doesn't conflict with future iterations.
3. `seed` target in `Makefile`:
   - Depends on `supabase-reset-database`.
   - Runs `npx tsx scripts/seed-supabase.ts`.
   - Uses the existing `run-silent` pattern is optional ‚Äî for the seeder we want stdout visible so failures are obvious.

## Test strategy

This iteration's tests are manual / runtime, exercised by `make seed` itself.

Surgical checks:
- `npx supabase status` runs cleanly (precondition for the seeder).
- `npx tsc --noEmit scripts/seed-supabase.ts` (or `make typecheck` if that picks it up).
- `make supabase-reset-database` then verify no rows in `companies`, `contacts`, `deals`, `tasks`, `tags`.
- `make seed` then verify exactly one row in `tags` with `name = '__seed_smoke_test__'`.
- `SUPABASE_URL=https://remote.example.supabase.co npx tsx scripts/seed-supabase.ts` ‚Üí exits non-zero with the localhost guard message; no DB touched.
- `npx tsx scripts/seed-supabase.ts` run twice without a reset ‚Üí second run exits non-zero with the "DB not clean" message.

`make test` and `make typecheck` are the standing project tests ‚Äî both should remain green.
